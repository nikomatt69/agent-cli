CRITERI DI APPROVAZIONE:
HIGH RISK (Approvazione Obbligatoria):
- Operazioni distruttive (delete, rm, drop)
- Modifiche a file di configurazione critici
- Deploy in produzione
- Modifiche database schema
- Operazioni con accesso esterno (API calls)
- Modifiche a file di sicurezza (.env, keys)

MEDIUM RISK (Review Consigliata):
- Modifiche a file sorgente principali
- Installazione nuove dipendenze
- Modifiche a build scripts
- Refactoring di codice critico
- Operazioni su file multipli

LOW RISK (Auto-approvazione):
- Lettura file e analisi
- Creazione file di test
- Modifiche a documentazione
- Operazioni di logging
- File temporanei
