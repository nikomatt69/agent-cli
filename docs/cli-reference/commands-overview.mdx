### üß† Model Management

<AccordionGroup>
  <Accordion title="/model - Switch AI Models">
    Switch between different AI models or show current model.
    
    ```bash
    # Show current model
    /model
    
    # Switch to specific model
    /model claude-3-5-sonnet
    /model gpt-4
    /model gemini-pro
    /model ollama:llama2
    ```
    
    **Supported Providers:**
    - Anthropic: `claude-3-opus`, `claude-3-sonnet`, `claude-3-haiku`
    - OpenAI: `gpt-4`, `gpt-3.5-turbo`
    - Google: `gemini-pro`, `gemini-pro-vision`
    - Ollama: `ollama:model-name`
  </Accordion>
  
  <Accordion title="/models - List Available Models">
    Display all available models with their status and capabilities.
    
    ```bash
    /models
    ```
    
    **Output includes:**
    - Model name and provider
    - Availability status
    - Token limits
    - Current API key status
  </Accordion>
  
  <Accordion title="/set-key - Configure API Keys">
    Set API keys for different AI providers.
    
    ```bash
    /set-key anthropic sk-ant-api03-...
    /set-key openai sk-...
    /set-key google AIza...
    ```
    
    <Warning>
      API keys are encrypted and stored securely in your local configuration.
    </Warning>
  </Accordion>
</AccordionGroup>

### ‚öôÔ∏è Configuration Commands

<AccordionGroup>
  <Accordion title="/config - View Configuration">
    Display current NikCLI configuration settings.
    
    ```bash
    /config
    ```
    
    **Shows:**
    - Default AI provider and model
    - Security settings
    - Session preferences
    - Cache configuration
  </Accordion>
  
  <Accordion title="/debug - Debug Configuration">
    Debug API key configuration and test AI generation.
    
    ```bash
    /debug
    ```
    
    **Performs:**
    - API key validation
    - Connection testing
    - Model availability check
    - Configuration diagnostics
  </Accordion>
  
  <Accordion title="/temp - Set Creativity Level">
    Configure AI temperature (creativity/randomness level).
    
    ```bash
    # Show current temperature
    /temp
    
    # Set temperature (0.0 = focused, 2.0 = creative)
    /temp 0.7
    /temp 1.2
    ```
    
    **Range:** 0.0 (most focused) to 2.0 (most creative)
  </Accordion>
  
  <Accordion title="/history - Chat History Control">
    Enable or disable chat history persistence.
    
    ```bash
    # Show current status
    /history
    
    # Enable/disable history
    /history on
    /history off
    ```
  </Accordion>
  
  <Accordion title="/system - System Prompt">
    Set or view the system prompt for the current session.
    
    ```bash
    # Show current system prompt
    /system
    
    # Set custom system prompt
    /system "You are a senior React developer specializing in TypeScript"
    ```
  </Accordion>
</AccordionGroup>

### üïê Session Management

<AccordionGroup>
  <Accordion title="/new - Start New Session">
    Create a new chat session with optional title.
    
    ```bash
    # Start new session
    /new
    
    # Start with title
    /new "React Authentication Project"
    /new "API Development"
    ```
  </Accordion>
  
  <Accordion title="/sessions - List Sessions">
    Display all saved chat sessions.
    
    ```bash
    /sessions
    ```
    
    **Shows:**
    - Session ID and title
    - Creation date
    - Message count
    - Last activity
  </Accordion>
  
  <Accordion title="/export - Export Session">
    Export session history to markdown format.
    
    ```bash
    # Export current session
    /export
    
    # Export specific session
    /export abc123
    ```
    
    **Output:** Markdown file with complete conversation history
  </Accordion>
  
  <Accordion title="/stats - Usage Statistics">
    Show usage statistics and metrics.
    
    ```bash
    /stats
    ```
    
    **Includes:**
    - Token usage by provider
    - Session counts
    - Command usage frequency
    - Time spent in different modes
  </Accordion>
</AccordionGroup>

### üë• Agent Management

<AccordionGroup>
  <Accordion title="/agents - List Available Agents">
    Display all available AI agents and their capabilities.
    
    ```bash
    /agents
    ```
    
    **Shows:**
    - Agent name and specialization
    - Current status (active/idle)
    - Capabilities and tools
    - Performance metrics
  </Accordion>
  
  <Accordion title="/agent - Run Specific Agent">
    Execute a specific agent with a given task.
    
    ```bash
    /agent coding-agent "analyze this function for optimization"
    /agent react-expert "create a responsive navigation component"
    /agent backend-agent "implement user authentication API"
    ```
  </Accordion>
  
  <Accordion title="/auto - Autonomous Execution">
    Enable autonomous multi-agent execution for complex tasks.
    
    ```bash
    /auto "Create a complete React todo app with backend API"
    /auto "Implement user authentication with JWT tokens"
    /auto "Add real-time chat functionality to the application"
    ```
    
    **Features:**
    - Automatic task breakdown
    - Multi-agent coordination
    - Minimal user intervention
    - Progress tracking
  </Accordion>
  
  <Accordion title="/parallel - Parallel Agent Execution">
    Run multiple agents simultaneously on the same task.
    
    ```bash
    /parallel "coding-agent,react-expert" "create a login component"
    /parallel "backend-agent,frontend-agent" "implement user profile management"
    ```
  </Accordion>
  
  <Accordion title="/factory - Agent Factory Dashboard">
    Open the agent factory for creating and managing custom agents.
    
    ```bash
    /factory
    ```
    
    **Provides:**
    - Agent creation wizard
    - Template library
    - Performance analytics
    - Agent deployment tools
  </Accordion>
  
  <Accordion title="/create-agent - Create Custom Agent">
    Create a new specialized agent for specific tasks.
    
    ```bash
    /create-agent "React Testing Expert"
    /create-agent "Database Optimization Specialist"
    /create-agent "UI/UX Design Assistant"
    ```
  </Accordion>
  
  <Accordion title="/launch-agent - Launch from Blueprint">
    Launch an agent from a saved blueprint configuration.
    
    ```bash
    /launch-agent abc123 "optimize performance"
    /launch-agent react-blueprint "create component library"
    ```
  </Accordion>
  
  <Accordion title="/context - Workspace Context">
    Select or view workspace context paths for agents.
    
    ```bash
    # Show current context
    /context
    
    # Set context paths
    /context src/ tests/
    /context src/components/ src/hooks/
    ```
    
    **Purpose:** Focuses agents on specific parts of your codebase
  </Accordion>
  
  <Accordion title="/stream - Agent Stream Dashboard">
    View live agent activity streams or clear completed streams.
    
    ```bash
    # Show stream dashboard
    /stream
    
    # Clear completed streams
    /stream clear
    ```
  </Accordion>
</AccordionGroup>

### üìã Planning & Todo Commands

<AccordionGroup>
  <Accordion title="/plan - Execution Planning">
    Create, execute, and manage execution plans for complex tasks.
    
    ```bash
    # Create new plan
    /plan create "Build authentication system"
    
    # Execute plan
    /plan execute [planId]
    
    # Show plan status
    /plan show [planId]
    
    # List all plans
    /plan list
    
    # Request approval
    /plan approve [planId]
    ```
    
    **Plan Features:**
    - Automatic task breakdown
    - Dependency tracking
    - Progress monitoring
    - Approval workflows
  </Accordion>
  
  <Accordion title="/todo - Todo Management">
    Manage todo lists and task tracking.
    
    ```bash
    # List all todo lists
    /todo list
    
    # Show todos for specific plan
    /todo show [planId]
    
    # Open todo.md in editor
    /todo open
    
    # Edit todo.md file
    /todo edit
    ```
  </Accordion>
  
  <Accordion title="/todos - Quick Todo List">
    Alias for `/todo list` - quickly view all todo items.
    
    ```bash
    /todos
    ```
  </Accordion>
</AccordionGroup>

### üõ°Ô∏è Security Commands

<AccordionGroup>
  <Accordion title="/security - Security Management">
    Manage security settings and view current security status.
    
    ```bash
    # Show security status
    /security status
    
    # Change security settings
    /security set approval-mode strict
    /security set command-execution restricted
    
    # Show security help
    /security help
    ```
    
    **Security Levels:**
    - **Safe Mode**: Maximum restrictions
    - **Default Mode**: Balanced approach
    - **Developer Mode**: Minimal restrictions
  </Accordion>
  
  <Accordion title="/dev-mode - Developer Mode">
    Enable developer mode with reduced security restrictions.
    
    ```bash
    # Enable for 30 minutes (default: 60)
    /dev-mode enable 30
    
    # Check status
    /dev-mode status
    
    # Show help
    /dev-mode help
    ```
    
    <Warning>
      Developer mode reduces security restrictions. Use with caution in production environments.
    </Warning>
  </Accordion>
  
  <Accordion title="/safe-mode - Maximum Security">
    Enable safe mode with maximum security restrictions.
    
    ```bash
    /safe-mode
    ```
    
    **Enables:**
    - Approval required for all operations
    - Restricted command execution
    - Limited file system access
    - Enhanced audit logging
  </Accordion>
  
  <Accordion title="/clear-approvals - Clear Session Approvals">
    Clear all cached approvals for the current session.
    
    ```bash
    /clear-approvals
    ```
    
    **Effect:** Requires fresh approval for previously approved operations
  </Accordion>
</AccordionGroup>

## Operating Modes

NikCLI supports three primary operating modes accessible via `Shift+Tab`:

  <Tab title="üí¨ Default Chat">

    - Step-by-step guidance

  <Tab title="üìã Planning Mode">
    **Strategic Planning**
    - Comprehensive execution plans
    - Todo.md file generation
    - Approval workflows
    - Best for complex projects
  </Tab>
  
  <Tab title="ü§ñ Auto Mode">

    - Automatic task execution
    - Minimal user intervention

## Keyboard Shortcuts

| Shortcut | Description |
|----------|-------------|
| `Ctrl+C` | Stop current operation or streaming response |
| `/` | Open command menu (when not processing) |
| `Shift+Tab` | Cycle between operating modes |
| `ESC` | Return to default chat mode |
| `Tab` | Auto-complete commands and file paths |
