### `/agent` - Individual Agent Execution

Execute tasks with specific agents, providing fine-grained control over agent selection and behavior.

<AccordionGroup>
  <Accordion title="Direct Agent Invocation">
    ```bash
    # Execute with universal agent
    /agent universal-agent "create a React login component"
    
    # Execute with specialized agent
    /agent react-expert "optimize this component for performance"
    /agent backend-agent "create REST API for user management"
    /agent devops-agent "set up CI/CD pipeline for deployment"
    ```
    
    **Agent Selection Guide:**
    - `universal-agent`: Best for general tasks requiring multiple skills
    - `react-expert`: Optimal for React/Next.js specific tasks
    - `backend-agent`: Ideal for API, database, and server-side logic
    - `devops-agent`: Perfect for deployment, monitoring, and infrastructure
  </Accordion>
  
  <Accordion title="Agent Configuration">
    ```bash
    # Configure agent parameters
    /agent react-expert --temperature 0.3 "refactor component"
    /agent backend-agent --max-tokens 4000 "design database schema"
    
    # Set agent context
    /agent universal-agent --context "src/components/" "analyze code"
    /agent devops-agent --workspace "infrastructure/" "update configs"
    
    # Agent execution modes
    /agent react-expert --mode creative "design new UI pattern"
    /agent backend-agent --mode conservative "fix security issue"
    ```
  </Accordion>
  
  <Accordion title="Agent Chaining">
    ```bash
    # Sequential agent execution
    /agent backend-agent "create API endpoint" | 
    /agent react-expert "create UI component for API"
    
    # Conditional agent chaining
    /agent universal-agent "analyze project structure" --then-if-frontend 
    /agent react-expert "optimize components"
    
    # Agent handoff with context
    /agent backend-agent "implement authentication" --handoff-to 
    /agent react-expert --context "auth implementation"
    ```
  </Accordion>
</AccordionGroup>

### `/parallel` - Parallel Agent Execution

Execute multiple agents simultaneously on related or independent tasks.

<AccordionGroup>
  <Accordion title="Parallel Task Distribution">
    ```bash
    # Parallel execution with multiple agents
    /parallel "react-expert,backend-agent" 
              "create user dashboard with API integration"
    
    # Distribute tasks across agents
    /parallel --distribute "optimize frontend performance" 
              --agents "react-expert,universal-agent"
    
    # Parallel with different tasks
    /parallel --tasks "frontend:react-expert:create UI components"
                     "backend:backend-agent:implement API endpoints"
                     "devops:devops-agent:setup deployment pipeline"
    ```
  </Accordion>
  
  <Accordion title="Parallel Coordination">
    ```bash
    # Coordinated parallel execution
    /parallel --coordinated 
              --agents "react-expert,backend-agent"
              "build authentication system"
    
    # Parallel with synchronization points
    /parallel --sync-points "design,implementation,testing"
              --agents "universal-agent,react-expert"
              "create component library"
    
    # Parallel with conflict resolution
    /parallel --resolve-conflicts auto
              --agents "react-expert,universal-agent"
              "refactor application architecture"
    ```

### `/launch-agent` - Blueprint-Based Agent Deployment

Deploy agents from pre-configured blueprints or saved configurations.

<Tabs>
  <Tab title="Blueprint Deployment">
    ```bash
    # Launch from saved blueprint
    /launch-agent blueprint-id "implement user authentication"
    
    # Launch with configuration override
    /launch-agent react-testing-blueprint 
                  --override temperature=0.2
                  "create comprehensive test suite"
    
    # Launch multiple agents from blueprint
    /launch-agent full-stack-blueprint
                  --agents "frontend,backend,devops"
                  "build complete application"
    ```
  </Tab>
  
  <Tab title="Blueprint Management">
    ```bash
    # List available blueprints
    /launch-agent --list-blueprints
    /launch-agent --blueprints --categories
    
    # Create blueprint from existing agent
    /launch-agent --save-blueprint agent-id "custom-react-expert"
    
    # Update blueprint configuration
    /launch-agent --update-blueprint blueprint-id
                  --add-skill "testing"
                  --modify-context "src/components/"
    ```
  </Tab>
  
  <Tab title="Blueprint Sharing">
    ```bash
    # Export blueprint for sharing
    /launch-agent --export-blueprint react-expert-v2
                  --format json --include-training-data
    
    # Import shared blueprint
    /launch-agent --import-blueprint ./shared-agent-blueprint.json
                  --validate --test
    
    # Blueprint version control
    /launch-agent --version-blueprint react-expert
                  --tag "v1.2.0" --changelog "Added hook optimization"
    ```

    ```bash
                --phases "design,backend,frontend,testing,deployment"
    ```

## Troubleshooting Agent Issues

### Common Agent Problems

<AccordionGroup>
  <Accordion title="Agent Not Responding">
    **Problem:** Agent becomes unresponsive or fails to complete tasks
    
    **Diagnostic Steps:**
    ```bash
    # Check agent health
    /agents health react-expert
    
    # View agent logs
    /debug agent react-expert --logs --recent
    
    # Check resource usage
    /agent-performance react-expert --resources
    ```
    
    **Solutions:**
    ```bash
    # Restart agent
    /agents restart react-expert
    
    # Reset agent state
    /agents reset react-expert --preserve-learning
    
    # Agent resource optimization
    /optimize-agents react-expert --memory --context
    ```
  </Accordion>
  
  <Accordion title="Poor Agent Performance">
    **Problem:** Agent produces low-quality outputs or takes too long
    
    **Diagnostic Steps:**
    ```bash
    # Analyze performance metrics
    /agent-performance react-expert --detailed
    
    # Check recent tasks
    /agents activity react-expert --recent --quality-analysis
    
    # Review agent configuration
    /agents info react-expert --config --capabilities
    ```
    
    **Solutions:**
    ```bash
    # Optimize agent configuration
    /optimize-agents react-expert --auto-tune
    
    # Update agent training
    /agent-learning update react-expert --recent-feedback
    
    # Adjust agent parameters
    /agents configure react-expert --temperature 0.3 --max-tokens 3000
    ```
  </Accordion>
  
  <Accordion title="Agent Conflicts">
    **Problem:** Multiple agents produce conflicting outputs or approaches
    
    **Diagnostic Steps:**
    ```bash
    # Analyze conflict patterns
    /orchestrate conflicts --agents "react-expert,universal-agent"
    
    # Review collaboration history
    /agent-collaboration history --conflicts --resolution
    ```
    
    **Solutions:**
    ```bash
    # Configure conflict resolution
    /orchestrate configure --conflict-resolution expert-priority
    
    # Improve agent coordination
    /agent-collaboration optimize --coordination-patterns
    
    # Establish agent hierarchies
    /agents hierarchy --senior react-expert --junior universal-agent
    ```
  </Accordion>
</AccordionGroup>
